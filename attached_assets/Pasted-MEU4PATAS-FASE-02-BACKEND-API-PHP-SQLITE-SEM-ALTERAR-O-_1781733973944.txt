4. CONFIGURAÇÃO PDO

⸻

Criar config/connection.php.

Regras:

* Usar PDO com SQLite.
* Caminho principal: DIR . ‘/../banco.db’
* Ativar erros como exceção.
* Ativar foreign keys sempre.
* Retornar JSON em caso de erro.

Exemplo obrigatório:

$pdo = new PDO(‘sqlite:’ . $dbPath);
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
$pdo->setAttribute(PDO::ATTR_DEFAULT_FETCH_MODE, PDO::FETCH_ASSOC);
$pdo->exec(“PRAGMA foreign_keys = ON;”);

Não deixar display_errors ativo para o usuário final em produção.

6. API DE ADOTANTES

⸻

Criar api/adotantes.php.

Ela deve retornar JSON e aceitar métodos:

GET:

* Listar todos os adotantes.
* Buscar um adotante por id, se vier ?id=1.

POST:

* Criar novo adotante.
* Receber JSON ou FormData.
* Validar:
    * nome_completo obrigatório
    * telefone obrigatório
    * endereco obrigatório
    * CPF válido, se enviado
    * email válido, se enviado
    * idade >= 21, se for para adoção

PUT:

* Atualizar adotante existente.
* Receber id.
* Validar dados.

DELETE:

* Excluir adotante.
* Antes de excluir, tratar pets vinculados usando ON DELETE SET NULL.

Resposta padrão:

{
“success”: true,
“message”: “…”,
“data”: …
}

Em erro:

{
“success”: false,
“message”: “…”
}

Usar prepared statements em todas as queries.

7. API DE PETS

⸻

Criar api/pets.php.

Métodos:

GET:

* Listar todos os pets.
* Separar ou permitir filtro por status.
* Permitir busca por nome, espécie, cidade, raça.
* Trazer nome do adotante quando pet estiver adotado usando JOIN.

A query de listagem precisa usar LEFT JOIN:

SELECT
pet.*,
adotante.nome_completo AS nome_adotante,
adotante.telefone AS telefone_adotante
FROM pet
LEFT JOIN adotante ON adotante.id = pet.adotante_id
ORDER BY pet.id DESC;

POST:

* Criar pet.
* Pet novo deve entrar inicialmente como Disponível.
* adotante_id deve ser NULL.
* Usuário comum não pode criar pet como Adotado.
* Validar:
    * nome_pet obrigatório
    * especie Cão ou Gato
    * idade_aproximada obrigatória
    * status Disponível ou Adotado apenas no backend administrativo
    * imagem opcional ou obrigatória conforme o front-end atual

PUT:

* Atualizar pet.
* Permitir alterar dados do pet.
* Não permitir inconsistência:
    * se status = Adotado, precisa ter adotante_id.
    * se status = Disponível, adotante_id deve ser NULL.

DELETE:

* Excluir pet.

Todos os métodos devem retornar JSON.

* adotante_id obrigatório
