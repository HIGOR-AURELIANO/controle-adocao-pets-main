MEU4PATAS — IMPLEMENTAR BACKEND REAL PHP + SQLITE NO REPLIT

Você vai corrigir e finalizar o projeto meu4patas para rodar no Replit usando PHP 8, SQLite, PDO e Fetch API.

Contexto:
O projeto já possui front-end moderno com:

* index.html
* cadastro.html
* cadastrar-pet.html
* perfil.html
* css/style.css
* js/script.js ou js/Script.js
* assets/
* uploads/

A interface atual deve ser preservada. Não reconstrua o layout do zero.

O problema atual é que o projeto está misturado:

* HTML novo do meu4patas.
* Resquícios antigos de “Patinhas”.
* Banco antigo chamado patinhas.sqlite.
* JS antigo chamando arquivos como api/listar_pets.php e api/adotar_pet.php.
* Estrutura ainda sem integração final com SQLite via PDO.

Objetivo:
Criar a versão final funcional para Replit, usando:

* PHP 8
* SQLite
* PDO
* api.php
* banco.db
* Fetch API
* CRUD completo de pets e adotantes
* Cadastro/login de usuário
* Upload de imagem dos pets
* Interesse em pets
* Confirmação de adoção apenas por painel de gestão

IMPORTANTE:
O usuário comum NÃO realiza adoção direta.
O usuário comum apenas demonstra interesse.
O pet só muda para “Adotado” no painel de gestão/admin/responsável.

4. API.PHP

⸻

Criar api.php como API principal do sistema.

Toda resposta deve ter:

header(‘Content-Type: application/json; charset=utf-8’);

Formato de sucesso:

{
“success”: true,
“message”: “Operação realizada com sucesso”,
“data”: {}
}

Formato de erro:

{
“success”: false,
“message”: “Mensagem clara do erro”,
“data”: null
}

Regras:

1. Usar PDO.
2. Usar prepared statements.
3. Usar session_start().
4. Usar try/catch.
5. Validar dados no PHP.
6. Não retornar HTML pela API.
7. Não deixar warning quebrar JSON.
8. Ativar chaves estrangeiras:
    PRAGMA foreign_keys = ON;
